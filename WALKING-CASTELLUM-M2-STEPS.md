# M2 — The Map contract: execution steps

Expanded from `WALKING-CASTELLUM-PLAN.md` M2. Reference behavior derives from the plan's
Context/reuse map, ADR-0001, and ADR-0005. Exit gate: `make ci` green; determinism
snapshot identical; map renders from the runtime Map; no module-constant map imports
remain in `src/components`.

Shape decisions taken from the ADRs (no new user decisions):

- Connections are per-kind instances (ADR-0005): one `gas_line` or `liquid_line`
  connection per line, plus today's portals as architectural connection kinds. The
  paired `TransportRunDefinition` is retired; canonical process-line ids are
  `gas:a__b` / `liquid:a__b` with the room pair sorted lexicographically, generated by
  a shared helper so M3's player-built lines mint identical ids.
- Iteration order is behavior: gas/liquid flow and architectural flow resolve in
  today's authored order. The authored Map script must insert connections so that the
  kind-filtered order equals today's `TRANSPORT_RUNS` key order (per phase) and
  `FACILITY_MAP.portals` array order; a characterization test locks this.
- `ScenarioAvailability.gasRuns/liquidRuns` rename to `gasLines/liquidLines` ("run"
  is reserved for the roguelite run in the plan vocabulary); `TransportRunId` becomes
  `ConnectionId`. `set_conduit`/`build_transport`/`dismantle_transport` keep their
  names but address a `connectionId` and drop the redundant `phase` field (kind
  implies it). Old saves are cut off by the contentVersion bump (pre-release gate).
- Sources/buffers/rooms/equipment stay pack catalogs; the Map owns where things are
  (bounds, cells, routes, taps, utility-node locations) and what the world's topology
  is. Room taps (capacity, includeRoomInventory, portHeight, sourceIds) replace
  `GAS_JUNCTIONS`/`LIQUID_JUNCTIONS`.

1. The Map type and canonical id helpers
   - File(s): `src/game/world/map.ts` (new), `src/game/world/map.test.ts` (new)
   - Reference behavior: ADR-0001 (Map = grid + rooms + connections; provenance is
     plain data), ADR-0005 (connection kinds and fields). Field shapes come from
     today's `FacilityMapDefinition`, `FacilityPortalDefinition`,
     `ConduitPhaseDefinition`, `GasJunctionDefinition`/`LiquidJunctionDefinition`.
   - Change: `WorldMap` interface — grid header (width, height, cellSize, coreAnchor,
     ringRadii, entryCell, coreBreachCell), `rooms: Record<RoomId, MapRoom>` (bounds,
     socketCells, platformCells, ladderCells, `taps: { gas, liquid }`, `provenance:
"site" | "hull"`), `connections: Record<ConnectionId, MapConnection>` — a
     discriminated union: architectural kinds (`passage`, `ladder_shaft`,
     `floor_hole`, `door`, `trapdoor`, `core_door`) carrying today's portal fields,
     process kinds (`gas_line`, `liquid_line`) carrying direction, destinationKind,
     actuator, actuatorHead, maxFlow, volumePerCell, buildCost, route — plus
     `utilityNodes`. Helpers `processLineId(kind, a, b)` (sorted pair) and kind
     predicates (`isProcessLine`, `isArchitectural`).
   - Verify: red — module does not exist (test fails to resolve); green — id helper
     yields the same id regardless of argument order and distinct ids per kind.

2. Shared map-validation module
   - File(s): `src/game/world/mapValidation.ts` (new),
     `src/game/world/mapValidation.test.ts` (new)
   - Reference behavior: generalized from `authoring/compiler.ts` route checks
     (≥2 cells, in-bounds, orthogonal adjacency, endpoints inside their rooms) and
     record-key identity, plus ADR-0005's pair invariant (at most one gas and one
     liquid line per room pair). Operates on a `WorldMap` alone; issue list format
     mirrors `AuthoringIssue`.
   - Change: `validateWorldMap(map): readonly MapIssue[]` covering: record key ==
     declared id (rooms, connections), route validity for process lines against room
     bounds, per-pair line uniqueness, connection endpoints referencing known rooms,
     utility nodes hosted by known rooms, room taps' sourceIds non-duplicated.
   - Verify: red — synthetic bad maps (diagonal route, route ending outside its room,
     duplicate gas line on a pair, key/id mismatch) each produce their issue before
     the module exists (fails to resolve); green — all listed cases return the
     expected issue and today's authored map (step 3) returns none.
